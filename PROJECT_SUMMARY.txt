QUICK START
================================================================================

1. INSTALL DEPENDENCIES
   Ubuntu/Debian:
     sudo apt install python3-gi gir1.2-gtk-4.0 gir1.2-adwaita-1 \
       meson ninja-build python3-pip mpv

2. INSTALL PYTHON PACKAGES
   cd ~/Documents/ani-gui
   pip install --user -r requirements.txt

3. BUILD & INSTALL
   meson build --prefix=$HOME/.local
   meson compile -C build
   meson install -C build

4. RUN
   ani-gui

ALTERNATIVE (Development):
   cd ~/Documents/ani-gui
   ./run-dev.py

================================================================================
ARCHITECTURE OVERVIEW
================================================================================

Application Entry:
  main.py → AniGuiApplication (Gtk.Application)
         → Creates AniGuiWindow on activation

Main Window Layout:
  window.py (AniGuiWindow)
    ├── Header: Search bar
    ├── Content Pane:
    │   ├── Left: ScrolledWindow
    │   │   ├── Results box (AnimeSearchRow widgets)
    │   │   └── Pagination buttons
    │   └── Right: VideoPlayerWidget
    └── Status/Footer

Data Flow:
  SearchEntry → SearchHandler → AniListAPI → Results
                                  ↓
                            ThumbnailCache → Downloaded images
                                             Cached locally
  Result Click → VideoPlayerWidget → External player / Browser

Threading Model:
  Main Thread (GTK UI updates)
    ├─ Background Search Thread
    │   └─ GLib.idle_add() callback
    └─ Background Thumbnail Thread (per image)
       └─ GLib.idle_add() callback

================================================================================
KEY DESIGN PATTERNS
================================================================================

1. ASYNC/AWAIT PATTERN
   Search and thumbnail loading run in background threads to keep UI responsive

2. CALLBACK PATTERN
   GLib.idle_add() ensures all UI updates happen on main thread safely

3. WIDGET COMPOSITION
   AnimeSearchRow encapsulates UI for single result with data

4. API CLIENT PATTERN
   Single AniListAPI instance manages all GraphQL communication

5. CACHE PATTERN
   ThumbnailCache provides unified interface for disk/network operations
